## 修复：打工模拟器 Web (WASM) 版 — 彻底解决

**问题：**
1. 网页卡在"加载中..." — 用户直接双击 HTML (file:// 协议)
2. 频繁弹出 "Input:" 对话框 + alert — Emscripten 默认用 `window.prompt` 模拟 stdin

**方案 B：修改 C++ 源码 + 重新编译 WASM（根因解决）**

### C++ 改动（针对 `work_web.cpp`，`打工模拟器.cpp` 已还原为原始版）
- **`get_valid_input()`**: printf 移出 while，fgets NULL 时 sleep(100)
- **`get_valid_input_with_cancel()`**: 同上
- **HTML**: customStdin 返回 null (EOF)；去除 input echo (❯ 前缀)

### HTML 改动 (`打工模拟器.html`)
- 完整重写终端风格 UI（黑底、彩色 ANSI 渲染、圆角窗口）
- 修复 HTML 标签被 `ansiToHtml` 转义的问题：拆分 `writeOutput()` 和 `writeHtml()`
- 三层输入系统，**完全移除 window.prompt**：
  - Tier 1: 全局 keydown 捕获（加载动画 yield 期间收按键）
  - Tier 2: 底部输入框提交
  - Tier 3: customStdin 队列为空时返回 '\n'，C++ 自行 yield 等用户
- 自动推送初始 '\n' 跳过开头 press_any_key
- 过滤 Emscripten 无害告警

### 编译
- 使用用户已有的 `G:\emsdk\upstream\emscripten\emcc.exe`（v6.0.1）
- 命令：`emcc 打工模拟器.cpp -o 打工模拟器.js -s WASM=1 -s ASYNCIFY=1 -O2 -s NO_EXIT_RUNTIME=1`
- WASM 从 349KB → 190KB，JS 从 202KB → 66KB（新编译器优化更好）

## 继续修复：输入无反应 + ANSI/emoji 格式乱码

**问题：** 用户重新编译后 HTML 被覆盖（已还原），输入仍不工作，`[2J[H` 等转义码和 emoji 显示为乱码

**根因：** `打工模拟器.js` 中 `FS_stdin_getChar` 硬编码 `window.prompt("Input: ")`，且 ANSI 控制序列未被正确过滤

**修复：**
- **patch 打工模拟器.js**：将 `window.prompt("Input: ")` 替换为 `__inputQueue` 队列读取逻辑（索引定位替换）
- **HTML ansiToHtml**：增强处理 `[2J`（清屏→暗色分隔线）、`[H`（光标归位→移除）、`[1m`（粗体→CSS class）
- **HTML print**：改为 `writeOutput(text + '\n')`（Emscripten 的 print 不带换行）
- **HTML**：暴露 `window.__inputQueue = inputQueue` 供给 JS 胶水代码
- **HTML**：增加 `Module.onRuntimeInitialized` 覆盖 FS_stdin_getChar 作为双重保险
- CSS 字体栈添加 emoji 字体 `'Segoe UI Emoji', 'Apple Color Emoji'`
